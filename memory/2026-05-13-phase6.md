# memory/2026-05-13 (Phase 6 — responsive layout)

## Features Shipped
- **Viewport hook** (`hooks/useViewport.ts`): reports `{ w, h, breakpoint, isMobile, isMobilePortrait, isMobileLandscape, isTablet, isDesktop }`. Breakpoints: `mobile-portrait` (<768 portrait), `mobile-landscape` (<768 landscape), `tablet` (768-1279), `desktop` (≥1280). rAF-debounced on resize + orientationchange.
- **Dynamic seat layout** (`utils/seatLayout.ts`): replaced the static 9-slot SEAT_POSITIONS + getRelativeSeatPositions with `computeSeatPositions(seats, mySeatIndex, opts)` and `computeSeatPositionsForViewport`. Distributes seats evenly around a parametric oval. Local player always pinned bottom-centre, clockwise from there (matches real poker flow — SB sits to your right). ax/ay tuned per breakpoint so the oval tightens on narrow widths.
- **PokerTable.tsx** now consumes `useViewport()` and renders all cards / chip badges / position badges / name plates / pot pill at viewport-aware sizes. Action bar repositions to `position:fixed` bottom on mobile with `safe-area-inset-bottom` padding and 44px minimum touch targets.
- **PokerTableMobile.tsx** (new): stacked layout for mobile portrait. Opponents in a horizontal scroll-snap strip at top, then community-cards/pot felt strip, then your seat (avatar + name + stack + hole cards in one row), then sticky bottom action buttons.
- **GameRoom.tsx** branches: `viewport.isMobilePortrait ? <PokerTableMobile/> : <PokerTable/>`. DealAnimation only mounts on the oval path (mobile has its own hand-start visual cues). Container padding shrunk on mobile.

## Decisions
- Mobile portrait gets a *separate component*, not a CSS-only adaptation. The oval layout simply doesn't make sense at <600px wide for >2 players.
- Mobile landscape still uses the oval (tighter ax/ay) — works fine on iPhone landscape.
- DealAnimation reused unchanged; its `seatPositionByIndex` map is now built from `computeSeatPositionsForViewport` instead of `SEAT_POSITIONS[positionIndex]`.
- Pre-existing TS strict-mode warnings left unfixed (would need a wider refactor; vite build is unaffected).
- Front-end vitest config not present yet; relied on backend test suite (138/138 PASS) + vite production build for regression confidence.

## Layout breakpoint contract
| Breakpoint | Width | PokerTable? | Action bar | Cards size |
|---|---|---|---|---|
| desktop | ≥1280 | oval | beneath felt | w-12/w-[72px] |
| tablet | 768-1279 | oval (tighter) | beneath felt | w-8/w-12 |
| mobile-landscape | <768 landscape | oval (tightest) | fixed bottom | w-7/w-10 |
| mobile-portrait | <768 portrait | **PokerTableMobile** stacked | fixed bottom | w-6/w-9 |
